# وثيقة متطلبات المنتج (PRD)
# ركاز — منصة حجز خبيرات التجميل

**الإصدار:** 1.3
**التاريخ:** مارس 2026
**الحالة:** MVP مكتمل (Frontend + Backend) — قيد التحسين والتوسع

---

## 1. نظرة عامة على المنتج

### 1.1 ملخص المنتج

**ركاز** منصة رقمية تُتيح للعميلات حجز خبيرات المكياج ومصففات الشعر بسهولة تامة، وتُمكّن المبدعات (المزودات) من إدارة أعمالهن وتنظيم جداول حجوزاتهن، مع لوحة تحكم للمشرف لإدارة المنصة بالكامل.

### 1.2 المشكلة التي يحلّها

- **العميلات:** صعوبة إيجاد خبيرة موثوقة وحجز موعد بسرعة دون الاتصال الهاتفي.
- **المبدعات:** غياب أداة متكاملة لإدارة الحجوزات والعملاء والخدمات وتتبع الأرباح.
- **المشرفون:** الحاجة إلى رؤية شاملة لأداء المنصة وعائداتها وإدارة مزودي الخدمة.

### 1.3 الجمهور المستهدف

| الفئة | الوصف |
|-------|-------|
| العميلات | نساء يبحثن عن خدمات تجميل احترافية في السعودية |
| المبدعات | خبيرات مكياج، مصففات شعر، متخصصات عناية بالبشرة والأظافر |
| المشرف | إدارة المنصة والإشراف على المزودين والإيرادات |

### 1.4 نموذج العمل

- عمولة **2%** تُضاف فوق سعر الخدمة وتُدفع من قِبل العميلة (Escrow)
- اشتراكات شهرية/سنوية للمبدعات: مجانية / أساسية (49 ريال/شهر) / برو (99 ريال/شهر)
- المبالغ محجوزة في Escrow حتى تأكيد اكتمال الخدمة من الطرفين
- الأرباح تُضاف لمحفظة المبدعة وتُسحب عبر IBAN

## 2. أهداف المنتج

### 2.1 أهداف الأعمال
1. بناء سوق موثوق لخدمات التجميل في السعودية
2. تحقيق إيرادات مستدامة عبر العمولات والاشتراكات
3. تمكين المبدعات من رقمنة أعمالهن وزيادة دخلهن

### 2.2 مؤشرات النجاح (KPIs)
- عدد الحجوزات المكتملة شهرياً
- معدل الاحتفاظ بالمبدعات النشطات
- متوسط تقييم المنصة (الهدف: ≥ 4.5)
- معدل تحويل الزيارات إلى حجوزات
- إجمالي الإيرادات (عمولات + اشتراكات)

## 3. المستخدمون والأدوار

### 3.1 العميلة (Client)

المستخدمة النهائية التي تبحث عن خدمات التجميل وتحجزها.

**الصلاحيات:**
- تصفح المبدعات والخدمات
- إنشاء الحجوزات واختيار الحي + دفع رسومها (شامل عمولة 2%)
- مراسلة المبدعة مباشرةً داخل التطبيق (إنشاء محادثة)
- تأكيد استلام الخدمة بعد اكتمالها
- تقييم الخدمات المكتملة
- إدارة حساب شخصي

### 3.2 المبدعة (Provider)

مزودة الخدمة (خبيرة مكياج / مصففة شعر / إلخ).

**الصلاحيات:**
- إدارة قائمة الخدمات وأسعارها
- قبول/رفض الحجوزات
- إدارة ملف المتجر وأحياء التغطية وأوقات العمل
- الرد على رسائل العملاء (دون إنشاء محادثة جديدة)
- تأكيد اكتمال الخدمة لإطلاق مبالغ Escrow
- إدارة المحفظة وطلب سحب الأرباح عبر IBAN
- إنشاء كوبونات خصم

### 3.3 المشرف (Admin)

إدارة المنصة والإشراف العام.

**الصلاحيات:**
- عرض جميع المبدعات وتوثيقهن أو إلغاء التوثيق
- حل النزاعات بين العملاء والمبدعات
- إدارة طلبات سحب الأرباح (موافقة / رفض)
- إدارة باقات الاشتراك الثلاث
- تتبع إيرادات المنصة والعمولات
- الاطلاع على تقارير الأداء الشامل

## 4. قصص المستخدم (User Stories)

### 4.1 قصص العميلة

```
كعميلة، أريد:
- البحث عن خبيرة مكياج بالتخصص أو المدينة
- تصفح خدمات المبدعة مع الأسعار والمدد
- قراءة تقييمات ومراجعات عميلات سابقات
- معرفة الأحياء التي تغطيها المبدعة قبل الحجز
- اختيار حيّي عند إنشاء الحجز
- رؤية تفصيل السعر (سعر الخدمة + عمولة 2%)
- حجز موعد بتحديد التاريخ والوقت المناسب
- الدفع إلكترونياً (مدى/فيزا/Apple Pay)
- مراسلة المبدعة مباشرةً داخل التطبيق
- تأكيد استلام الخدمة بعد اكتمالها
- تتبع حالة حجوزاتي
- إلغاء حجز واسترداد المبلغ (إن كان ضمن سياسة الإلغاء)
- كتابة تقييم بعد اكتمال الخدمة
```

### 4.2 قصص المبدعة

```
كمبدعة، أريد:
- عرض خدماتي بصور وأسعار واضحة
- استقبال إشعارات فورية بالحجوزات الجديدة
- قبول أو رفض الحجوزات مع إبلاغ العميلة
- تحديد الأحياء التي أغطيها (متعددة الاختيار)
- الرد على رسائل العميلات داخل التطبيق
- تأكيد اكتمال الخدمة لإطلاق مبالغ المحفظة
- معرفة رقم العميلة فقط بعد تأكيد الطرفين
- الاطلاع على رصيد المحفظة المتاح والمعلّق
- طلب سحب الأرباح عبر IBAN
- إدارة أوقات عملي اليومية بمرونة
- الاطلاع على قائمة عملائي والتواصل معهن
- إنشاء كوبونات خصم لاستقطاب عملاء جدد
- تتبع أرباحي الشهرية والسنوية
```

### 4.3 قصص المشرف

```
كمشرف، أريد:
- رؤية إجمالي الإيرادات والعمولات المحصّلة
- مراجعة قائمة المبدعات وتوثيقهن أو إلغاء التوثيق
- إدارة باقات الاشتراك الثلاث (مجانية/أساسية/برو)
- عرض النزاعات المفتوحة والبتّ فيها
- الموافقة أو رفض طلبات سحب الأرباح
- الاطلاع على تقارير أداء المنصة
```

## 6. نظام Escrow والمدفوعات

### 6.1 آلية الدفع

```
العميلة تدفع (سعر الخدمة + 2% عمولة)
    ↓
المبلغ محجوز في Escrow
    ↓
يوم الموعد: تكتمل الخدمة
    ↓
المبدعة تضغط "تأكيد اكتمال الخدمة"
    ↓
العميلة تضغط "تأكيد استلام الخدمة"
    ↓
(عند تأكيد الطرفين)
    ↓
المبلغ يُضاف لمحفظة المبدعة (متاح للسحب)
```

### 6.2 آلية حل النزاعات

- إذا لم تؤكد العميلة خلال 24 ساعة → يُنشأ نزاع تلقائياً
- المشرف يطّلع على بيانات الطرفين ويبتّ في النزاع
- القرار: صالح العميلة (استرداد) أو صالح المبدعة (إطلاق المبلغ)

### 6.3 الخصوصية

- رقم هاتف العميلة **مخفي** عن المبدعة حتى يؤكد الطرفان اكتمال الخدمة
- بعد التأكيد المزدوج: يظهر رقم العميلة للمبدعة عبر رابط واتساب

## 7. نظام الرسائل الداخلية

### 7.1 قواعد المراسلة

| الدور | يستطيع |
|-------|--------|
| العميلة | إنشاء محادثة جديدة مع مبدعة، إرسال رسائل |
| المبدعة | الرد على محادثات العملاء فقط (لا تُنشئ محادثات) |
| المشرف | لا يشارك في الرسائل |

### 7.2 واجهة الرسائل

- **العميلة:** تبويب "رسائلي" في التنقل السفلي
- **المبدعة:** قائمة "رسائل العملاء" في الإعدادات + شارة عدد غير المقروءة
- واجهة chat كلاسيكية مع ألوان مختلفة للطرفين

## 8. نظام المحفظة

### 8.1 حالات الرصيد

| الحالة | الوصف |
|--------|-------|
| متاح للسحب | مبالغ مكتملة ومؤكدة من الطرفين |
| معلّق | مبالغ حجوزات مؤكدة لم يتم تأكيد اكتمالها بعد |
| إجمالي الأرباح | تاريخي شامل (متاح + معلّق + مسحوب) |

### 8.2 السحب

- تُدخل المبدعة IBAN والمبلغ
- يصل الطلب للمشرف في لوحة التحكم
- المشرف يوافق أو يرفض

## 11. متطلبات غير وظيفية (Non-Functional Requirements)

### 11.1 الأداء
- وقت تحميل الصفحة الأولية: ≤ 2 ثانية (3G)
- وقت الاستجابة لطلبات API: ≤ 500ms
- تحمّل حتى 10,000 مستخدم متزامن

### 11.2 الأمان
- تشفير جميع البيانات الحساسة (HTTPS/TLS)
- تشفير كلمات المرور (bcrypt)
- JWT للمصادقة مع انتهاء صلاحية الجلسة
- التحقق من صحة جميع المدخلات على الخادم
- حماية من CSRF وXSS وSQL Injection
- تحديد معدل الطلبات (Rate Limiting) على نقاط API الحساسة

### 11.3 إمكانية الوصول
- دعم RTL كامل للغة العربية
- التوافق مع قارئات الشاشة (WCAG 2.1 AA)
- أحجام خطوط مقروءة على الشاشات الصغيرة (≥ 14px)
- تباين ألوان كافٍ (نسبة ≥ 4.5:1)

### 11.4 التوافق
- الهواتف الذكية: iOS 14+ و Android 9+
- المتصفحات: Chrome 90+، Safari 14+، Firefox 88+
- دقة الشاشة الأساسية: 390px (iPhone 14)

### 11.5 الموثوقية
- وقت التشغيل: ≥ 99.5%
- نسخ احتياطي يومي لقاعدة البيانات
- التعافي من الأعطال خلال ≤ 4 ساعات

## 14. سير العمل الرئيسية (Workflows)

### 14.1 رحلة إنشاء الحجز والدفع

```
العميلة تفتح التطبيق
    ↓
تتصفح المبدعات / تبحث
    ↓
تفتح ملف المبدعة (تطّلع على أحياء التغطية)
    ↓
تختار الخدمة → "احجزي الآن"
    ↓
تختار التاريخ والوقت والحي
    ↓
تراجع ملخص: سعر الخدمة + عمولة 2% = الإجمالي
    ↓
تدفع → المبلغ يُحجز في Escrow
    ↓
تُرسَل إشعار للمبدعة (PENDING)
    ↓
المبدعة تقبل → حالة: CONFIRMED
    ↓ (يوم الموعد)
المبدعة تؤكد الاكتمال + العميلة تؤكد الاستلام
    ↓
المبلغ يُطلق → يُضاف لمحفظة المبدعة
    ↓
العميلة تكتب تقييماً (اختياري)
```

### 14.2 رحلة النزاع

```
حجز مؤكد بعد 24 ساعة بدون تأكيد من العميلة
    ↓
إنشاء نزاع تلقائي (DISPUTED)
    ↓
المشرف يراجع بيانات الطرفين وأرقام الاتصال
    ↓
المشرف يكتب قرار الحل
    ↓
"صالح العميلة" → استرداد المبلغ
أو
"صالح المبدعة" → إطلاق المبلغ لمحفظتها
```

### 14.3 رحلة طلب السحب

```
المبدعة تفتح المحفظة → "سحب الأرباح"
    ↓
تُدخل IBAN + المبلغ
    ↓
الطلب يصل للمشرف (PENDING)
    ↓
المشرف يوافق → المبلغ يُحوَّل + يُسجَّل PAYOUT
أو
المشرف يرفض → إبلاغ المبدعة بالسبب
```
