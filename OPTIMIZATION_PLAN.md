## Remaining Backlog (Prioritized)

### Phase 1 ‚Äî Security & Reliability (Next Sprint, ~2 weeks)

**SEC-1: Validate service ownership on booking creation**
- Agent: Code Reviewer
- Risk: A client can book `serviceId=s1` with `providerId=p2` ‚Äî cross-provider mismatch not caught.
- Fix: After fetching the service in `POST /api/bookings`, verify `service.provider_id === providerId`.

**SEC-2: Add authentication to the .NET backend**
- Agent: Senior Architect + Senior Backend
- Risk: .NET API (port 5000) is fully open ‚Äî no token, no secret. Anyone knowing the URL can read/write bookings and wallets.
- Fix: Add a shared-secret middleware or JWT validation in `Ziena.API/Program.cs`. Minimal approach: require `X-Internal-Token` header matching an env var on all controllers.

**SEC-3: Race condition on wallet payment (double-credit)**
- Agent: Senior Backend
- Risk: Two simultaneous `PATCH /pay` calls on the same booking can both succeed and double-credit the provider wallet.
- Fix: Atomic conditional update:
  ```sql
  UPDATE bookings SET payment_status = 'PAID'
  WHERE id = ? AND payment_status = 'UNPAID'
  ```
  Check rows affected; if 0, reject with 409.

**REL-1: Commission rounding inconsistency between Node.js and .NET**
- Agent: Senior QA
- Risk: Node.js uses `Math.round(price * 0.02 * 100) / 100`; .NET uses `Math.Ceiling`. Results diverge on certain prices (e.g. Ô∑º333 ‚Üí Node: Ô∑º6.66, .NET: Ô∑º7).
- Fix: Standardize on `Math.ceil(price * 0.02)` (integer riyals, always rounds up) in both systems.

### Phase 2 ‚Äî Infrastructure & Observability (Month 2)

**INF-1: Add CI/CD pipeline**
- Agent: Senior DevOps
- No `.github/workflows/` exists ‚Äî every deploy is a manual push.
- Add: TypeScript check (`npm run lint`), .NET build (`dotnet build`), future test run on every push/PR.

**INF-2: Fix Dockerfile**
- Agent: Senior DevOps
- Current Dockerfile runs as root and has no HEALTHCHECK.
- Fix: Add `HEALTHCHECK CMD curl -f http://localhost:5000/health || exit 1` and switch to a non-root `app` user.

**INF-3: SQLite is ephemeral on Render free tier**
- Agent: Senior DevOps + CTO
- Risk: Every re-deploy wipes the .NET SQLite database ‚Äî all bookings and wallet data in .NET is lost.
- Fix options (pick one):
  1. Mount a Render Persistent Disk to `/data`, update connection string to `/data/ziena.db`
  2. Migrate .NET to Turso (same vendor as Node.js)
  3. Add a hosted PostgreSQL (Render free PostgreSQL tier)

**MON-1: Add error tracking and uptime monitoring**
- Agent: Senior DevOps + CTO
- Missing: No error tracking (Sentry), no uptime alerting, no request logging.
- Recommended: Sentry free tier for both Node.js and .NET; BetterUptime for availability pings.
